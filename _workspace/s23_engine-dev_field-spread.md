# s23 · engine-dev · 고밀도 세로(진행 방향) 필드 펴기

## 목표
고밀도(8·16명) 경주에서 캐치업(catchup)이 뒤처진 주자를 평균으로 당겨 다들 같은 progress에
뭉치는 문제 해결. 캐치업(뒤 당김)을 줄여 필드를 앞뒤로 늘어서게 하고, 독주는 막는다.
소규모(≤6명)는 기존 룩 유지.

## 핵심 발견 (설계가 바뀐 이유)
team-lead 설계는 "캐치업 완화 + **선두 핸디캡(리더 러버밴드)**"이었으나,
측정 결과 **선두 드래그 증폭(aheadBoost)이 역효과**였다:
- 16명/1랩 슬롯 측정에서 aheadBoost를 켜면 worst-slot 우승 수가 154 → 177~184로 치솟아
  **engine-bias 슬롯 천장(165)을 깨뜨림** (안쪽 유리 출발슬롯을 과보상).
- 동시에 선두를 평균으로 더 강하게 당겨 **앞쪽 팩이 다시 뭉쳐 spread도 줄어듦**.

즉 "앞을 누르는" 핸디캡 증폭은 공정성·펼침 둘 다 손해. 반면 **트레일러 테일윈드만 페이드**하면
공정성을 지키며 더 잘 펴진다. 기존 `aheadDrag=2.2`(대칭 캐치업의 선두 드래그)가 이미 충분한
anti-runaway 리더 러버밴드 역할을 하므로(no-runaway 게이트 통과), **추가 선두 핸디캡 없이**
"캐치업 뒤당김 완화"만으로 목표 달성.

## 구현 (엔진만)
- `src/engine/tuning.ts` — `CATCHUP.spread` 추가:
  활성주자 수가 `kneeAt`(6) 초과분마다 트레일러 테일윈드를 `behindFade`(0.06)씩 줄이고
  `behindMin`(0.4)에서 클램프. ≤6명이면 스케일 1 (소규모 룩 보존). 선두 드래그는 base 유지.
- `src/engine/RaceEngine.ts`:
  - `spreadBehindFor(active)` — 활성주자 수의 순수 함수(결정론적).
  - `internal.spreadBehind` — meanProgress와 함께 프레임당 1회 캐시(활성주자 수 = 쿨다운 knee와 동일 카운트).
  - `catchupFactor` — 트레일러 부스트에 `spreadBehind` 곱. 선두 드래그는 그대로.
- 무작위성 0 추가. progress 직접 점프 없음(속도 경로만). 결정론·엔진 순수성 유지.
