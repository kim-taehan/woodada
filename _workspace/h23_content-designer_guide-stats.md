# h23 content-designer — 캐릭터 가이드에 speed/power 표기

사용자: "speed/power 내용도 캐릭터 소개에 추가." 대상 = GuideOverlay가 렌더하는 `docs/character-guide.md`(설정화면 🐾캐릭터 탭). 마크다운만 수정, 코드 무관.

## 스탯값 확정
balance-tuner 확인: per-char 1~5 값 **불변·최종**(h22는 곰 roar params만, 독수리 4/2→3/3 시도 후 원복). 소스(src/data/characters/*.ts)와 일치 확인:
강아지 5/1, 고양이 4/2, 원숭이 3/3, 독수리 4/2, 펭귄 2/4, 곰 1/5, 고슴도치 2/4.

## 수정 (docs/character-guide.md만)
1. 도입부에 speed/power 설명 한 줄(blockquote): 속도=초반 페이스, 파워=몸싸움·추월 저항, 합 일정→공정.
2. "한눈에 보기" 표에 **🏃 속도 / 🛡️ 파워 컬럼** 추가 + 7행(값 표기).
3. 각 캐릭터 항목에 dot-bar 라인 추가: `- **🏃 속도 ●●●●○ 4 ／ 🛡️ 파워 ●●○○○ 2** — 한줄 설명`. 채운 ●가 값.
4. **고슴도치 항목 신규 추가**(기존 가이드에 누락돼 있었음 — 7종 표기 위해 필수): 소개/스킬(가시 반동 카운터)/dot-bar/기존 A~D 표 형식 맞춤.

## ⚠️ 함께 고친 것 / 발견한 것 (팀리드 확인)
- **독수리 prose가 stale였음**: "하늘에서 콱/트랙 위를 떠서 날아요/급강하 폭격" → 실제론 지상 폴짝 박치기로 전환된 캐릭터. dot-bar를 다는 그 항목이 거짓 설명을 담고 있어, 함께 **"폴짝 박치기/박치기(도박 스턴)"로 수정**(요약표 행 포함). 외과적 범위를 살짝 넘지만 같은 항목을 편집 중이고 사실과 모순돼 정정함.
- **이미지 누락/stale (마크다운 아님 → renderer-dev 영역)**:
  - `docs/img/hedgehog.png` **없음** → 내가 추가한 고슴도치 항목의 이미지가 안 뜸(깨진 img). renderer-dev가 렌더러 캡처로 생성 필요.
  - `docs/img/eagle.png` 는 **옛 비행 독수리** 캡처 → 날카로워진 지상 독수리로 재캡처 필요.

## 마크다운 호환성 확인 (src/shell/markdown.ts)
- 커스텀 렌더러는 **inline code(백틱) 미지원** → dot-bar에서 백틱 제거(●를 plain bold 텍스트로). 굵게/표/blockquote/리스트/이미지/hr만 지원되며 전부 사용 범위 내.
