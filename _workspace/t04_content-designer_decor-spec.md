# t04 — content-designer: decor / ambient 비주얼 스펙 (feat/arenas)

아트디렉션 문서. renderer-dev가 이 도형/색을 Pixi Graphics로 그린다(역할 분리: 나=아트디렉션, renderer=드로잉).
배치 좌표는 이미 테마 파일(`src/data/tracks/*.ts`)의 `DecorSpec.x/y/scale`에 있으니 건드리지 않는다 — **여기는 시각 스펙만**.

## 공통 가이드 (가독성 = 최우선)
- 소품은 **배경**이다. 트랙면/캐릭터를 가리거나 튀면 안 됨.
- `scale: 1` 기준 대략 크기: 정규화 좌표 기준 **폭 0.05~0.09, 높이 0.06~0.12 (씬 짧은 변 비율)** 정도. 즉 캔버스 한 변의 5~10% 수준의 작고 납작한 실루엣. scale은 이 기준을 곱한다.
- 톤: **채도·명도 낮춤**. 캐릭터(밝고 선명한 치비)보다 한 단계 죽은 색으로. 외곽선 쓰면 얇게(1~2px) 또는 생략. 둥근 모서리·납작한 비례로 귀엽게.
- alpha: 하늘·필드 영역 소품은 `alpha 0.85~1.0`. 멀리 있는 실루엣(building)은 0.7~0.9로 깔아 깊이감.
- 그림자/디테일 최소. 도형 2~4개 조합이면 충분.
- 좌표 `x,y`는 소품의 **앵커(대략 중심~하단 중앙)**. renderer가 일관되게 한 기준으로 잡으면 됨(권장: 바닥이 있는 것=하단중앙, 하늘에 뜬 것=중심).

## decor 12종 (테마당 2~3, 총 어휘 12)

### ☀️ sun (desert, beach)
- 도형: 채운 원 1개 + 짧은 직사각형/삼각형 광선 8~12개를 방사.
- 색: 본체 `0xffd24a`, 광선 `0xffe48a`(연하게). desert에선 더 쨍하게 본체 `0xffc233`.
- 크기: 본체 지름 ≈ 폭 0.06. 광선 포함 0.09. 납작하지 않은 정원.

### 🌙 moon (citynight)
- 도형: 채운 원 1개(초승달 원하면 같은 색 배경 원으로 일부 가림 = crescent). 간단히 보름달 원으로 가도 OK.
- 색: `0xf2eedb`(살짝 누런 흰), 외곽 글로우 `0xf2eedb alpha 0.25` 큰 원.
- 크기: 지름 ≈ 0.055. 야경이라 살짝 빛나게.

### ☁️ cloud (beach, snow / grassland은 미사용)
- 도형: 겹친 원 3~4개(가운데 큼·양옆 작음)로 뭉게 실루엣. 바닥은 평평하게(원들 중심 y 정렬).
- 색: `0xffffff` (snow 테마는 `0xf3f8fc`로 살짝 푸르게). alpha 0.92.
- 크기: 전체 폭 ≈ 0.08, 높이 ≈ 0.035 (납작·옆으로 길게).

### 🌵 cactus (desert)
- 도형: 세로 둥근 직사각형(기둥) + 양옆에 짧은 팔 1~2개(작은 둥근 막대, 위로 꺾임). 라운드 캡.
- 색: `0x5fa05a`(죽은 녹색), 살짝 어두운 외곽 `0x4d8a49`. 가시 표현 생략.
- 크기: 높이 ≈ 0.1, 기둥 폭 ≈ 0.02. 세로로 길쭉.

### 🌴 palm (desert, beach)
- 도형: 가는 줄기(살짝 휜 직사각/path) + 상단에 잎 4~6개(긴 타원 또는 삼각형이 방사).
- 색: 줄기 `0x9a6b3c`, 잎 `0x4f9e57`(살짝 죽은 녹). 잎 끝 약간 더 밝게 옵션.
- 크기: 높이 ≈ 0.12, 잎 스프레드 폭 ≈ 0.07.

### ⛱️ parasol (beach)
- 도형: 반원(돔, 위로 볼록) 캐노피 + 가는 세로 막대(기둥). 캐노피에 흰/색 교대 부채살 2~3개 선택.
- 색: 캐노피 `0xe9603f`(코랄, 살짝 죽임) + 흰 `0xfff4ec` 교대. 기둥 `0xccb89a`.
- 크기: 캐노피 폭 ≈ 0.07, 전체 높이 ≈ 0.08.

### 🛟 tube (beach)
- 도형: 도넛(큰 원 - 작은 원 구멍, 또는 stroke 두꺼운 링). 위에 흰 하이라이트 호.
- 색: 링 `0xff6f61`(연한 코랄) + 흰 `0xffffff` 4분할 교대(클래식 튜브) 선택. alpha 0.95.
- 크기: 지름 ≈ 0.05. 모래 위에 납작하게 눕힌 느낌(살짝 타원으로 눌러도 OK).

### 🏙️ building (citynight)
- 도형: 세로 직사각형(여러 높이로 겹쳐 스카이라인) + 창문 = 작은 점/사각 격자(2~3열).
- 색: 본체 `0x2b3050`(어두운 청회, 하늘보다 한 톤 밝게 → 실루엣 분리), 창문 `0xffd66b alpha 0.85`(켜진 창). alpha 본체 0.9.
- 크기: 폭 ≈ 0.05, 높이 0.08~0.16(변주). 하단을 화면 밖/필드 뒤로 깔아 스카이라인.

### 🌲 pine (snow)
- 도형: 겹친 삼각형 2~3개(아래로 갈수록 넓음) + 작은 줄기 사각형. 위에 흰 눈 캡(작은 호) 옵션.
- 색: 잎 `0x3f7a4e`(죽은 상록), 줄기 `0x6e4a2e`, 눈 `0xffffff`.
- 크기: 높이 ≈ 0.1, 밑변 폭 ≈ 0.05.

### ⛄ snowman (snow)
- 도형: 큰 원(하) + 작은 원(상) 2단 + 점 눈 2 + 주황 삼각 코 + (옵션) 막대 팔.
- 색: 몸 `0xfbfdff`(거의 흰, 외곽 `0xdfeaf2` 얇게), 코 `0xf08a3c`, 눈·단추 `0x333333`.
- 크기: 높이 ≈ 0.07, 폭 ≈ 0.045. 통통·귀엽게.

### 🍃 leaf (jungle)
- 도형: 큰 잎 1장 = 타원 + 중앙 잎맥(가는 선) + 좌우 측맥 2~3개. 또는 몬스테라식 path(절개) — 단순 타원으로도 충분.
- 색: `0x357a3f`(짙은 녹), 잎맥 `0x2a5f32`. 가장자리 살짝 밝게 `0x4a9152` 옵션.
- 크기: 폭 ≈ 0.08, 높이 ≈ 0.06. 화면 모서리에서 안쪽으로 드리운 듯 배치 가정.

### 🌿 vine (jungle)
- 도형: 위에서 늘어진 곡선(흔들리는 path/베지어) + 작은 잎 점 3~5개를 따라 붙임.
- 색: 줄기 `0x4a7a3a`, 잎 `0x3f8a46`. alpha 0.9.
- 크기: 폭 ≈ 0.02, 높이(드리움) ≈ 0.12. 화면 상단에서 아래로 늘어뜨림.

> 정글 fireflies(ambient)와 색이 겹치지 않게 vine/leaf는 녹색 계열로 통일, 발광은 ambient가 담당.

## ambient 파티클 4종

배경 파티클은 **희미하게**(저밀도·저알파) — 장식이지 정보가 아님. 트랙면 위에 떠도 캐릭터 판별을 방해 안 할 정도.

### sand (desert)
- 입자: 작은 가로 줄/점, 크기 1~2px. 색 `0xe9cf9b` alpha 0.35~0.5.
- 움직임: **수평 바람** — 한 방향(좌→우 권장)으로 흘러감, 약간의 상하 흔들림. 빠르고 성기게(20~30개).

### snow (snow)
- 입자: 둥근 점, 크기 2~4px(원근으로 변주). 색 `0xffffff` alpha 0.7~0.9.
- 움직임: 천천히 **아래로 낙하** + 좌우 사인 흔들림(나풀). 중밀도(30~50개).

### fireflies (jungle)
- 입자: 작은 글로우 점, 코어 2px + 부드러운 헤일로. 색 코어 `0xfff3a0`, 글로우 `0xd6ff7a alpha 0.4`.
- 움직임: 느리게 **떠다님**(랜덤 드리프트) + **깜빡임**(alpha 0.2↔0.9 펄스). 저밀도(12~20개), 주로 필드/하단 어둠 영역.

### petals (예약, 현재 미사용)
- 입자: 작은 타원 꽃잎. 색 `0xffc6d8` alpha 0.7.
- 움직임: 천천히 낙하 + 회전(빙글). 봄 테마 추가 시 사용. **지금은 구현 불필요**(스키마 예약만).

> ambient 결정론: 엔진/리플레이엔 무영향이지만, 스크린샷 골든 일관성을 위해 시각 파티클도 **테마/시간 기반 deterministic seed**(예: 프레임 t 함수)로 두면 캡처가 안정적. renderer-dev 재량.

## renderer-dev 인계 요약
- 도형 12 + ambient 3(+petals 예약)만 그리면 됨. 좌표/scale은 테마 데이터에서 옴.
- 핵심 원칙: 작고·납작·둥글게 + 채도/명도 한 단계 죽여 캐릭터보다 뒤로. 트랙면·캐릭터 가독성 최우선.
- citynight building 창문 색만 하이라이트(켜진 창)로 포인트 허용 — 나머지는 배경 톤 유지.

## 좌표 수정 — beach parasol (renderer-dev 질문 답, 데이터 반영 완료)
1차 구현에서 beach parasol y=0.70/0.74가 **하단 직선 트랙면**(y*800≈560~592) 위에 떨어져 "decor는 트랙면 위에 겹치지 않게" 규칙에 어긋났음. **외곽 모래밭(하단 직선 아래)로 내림** — content-designer가 `src/data/tracks/beach.ts` 데이터에 직접 반영:
- parasol #1: y 0.70 → **0.88** (x 0.14 유지, scale 1)
- parasol #2: y 0.74 → **0.90** (x 0.82 유지, scale 0.9)
근거: 트랙 바깥 surround=모래밭이라 파라솔이 자연스럽고 트랙면 가독성 회복. infield(y~0.5)는 물색(`0x36b5c4`)이라 파라솔엔 부적합 → 외곽 모래가 맞음. (tube는 y=0.5 infield 물 위 — 의도된 배치라 유지.)
renderer 코드 변경 불필요(데이터 좌표만). 다음 캡처 때 파라솔이 트랙 아래 모래밭에 깔립니다.
