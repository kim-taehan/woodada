## 4. 신규 decor kind 비주얼 스펙 (renderer-dev 인계)

렌더러 규약: `decor<Kind>(g: Graphics, x: number, y: number, s: number)`. (x,y)=픽셀(정규화×씬크기), s=scale. 좌표는 기준점(앵커) 기준. `DECOR_DRAW`에 등록. 기존 12 kind 형태/사이즈 톤(작은 소품 10~30px·s, 큰 소품 50~120px·s)과 맞춤. 모두 순수 시각·결정론(랜덤 금지). 가독성: 소품은 배경·외곽이라 알파 0.9 내외, 트랙면 위로 안 가지만 인필드 위 소품(flower/beachball/tube)은 또렷하게.

### 4.1 `bunting` — 만국기/깃발 가랜드 (grassland)
- **앵커**: (x,y) = 가랜드 좌측 시작점(상단). 가랜드는 오른쪽으로 늘어지며 줄이 살짝 처짐(sag).
- **구성**:
  - 처지는 줄: `(x,y)`에서 `(x + 240*s, y)`까지, 중앙이 `+18*s` 아래로 처지는 2차 베지어. stroke `0x8a6d3a`(끈색), width `2*s`, alpha 0.7.
  - 삼각 깃발 ~8개: 줄을 따라 등간격. 각 깃발은 줄의 점에서 아래로 향하는 이등변 삼각형, 폭 `16*s`·높이 `20*s`. 색은 운동회 팔레트 순환: `[0xe7574d(빨), 0xf5b13d(노), 0x4fa3e0(파), 0x5fbf6a(초)]` 인덱스 i%4. alpha 0.92.
- **크기 가이드**: 한 스팬 가로 ≈ 240*s px(씬 폭의 약 0.25). grassland에서 0.06·0.50 두 스팬이 상단을 가로지름.
- **가독성**: y≈0.04(최상단 스탠드/하늘) → 트랙·캐릭터와 안 겹침.

### 4.2 `flower` — 꽃 클럼프 (grassland)
- **앵커**: (x,y) = 클럼프 바닥 중심(땅에 핀 느낌).
- **구성**: 작은 꽃 3~4송이를 `±14*s` 범위로 흩뿌림. 각 꽃 = 꽃잎 5장(반지름 `4*s` 원, 중심에서 `5*s` 거리 방사) + 중앙 노란 원(`3*s`, `0xffd64a`). 꽃잎 색 교대 `0xff8fb3`(분홍)/`0xfff3a0`(연노)/`0xe88fe0`(연보라). 줄기/잎: 짧은 초록 stroke `0x4f9e57` width `2*s` 아래로 `6*s`.
- **크기 가이드**: 클럼프 폭 ≈ 30*s px. 인필드 정중앙(0.5,0.52)에 1개.
- **가독성**: 초록 인필드 위 → 분홍/노랑 꽃이 또렷. snowman과 같은 안전 중앙존.

### 4.3 `seagull` — 갈매기 (beach)
- **앵커**: (x,y) = 날개 중심(하늘 비행).
- **구성**: 갈매기 실루엣 = 갈매기형 두 곡선("ㅡ"가 가운데서 위로 꺾인 M 곡선). 좌우 날개 각각 2차 베지어로 `(x-18*s,y)`→정점`(x-9*s,y-7*s)`→`(x,y)` 및 대칭. stroke `0x55606e`(짙은 회색) width `3*s`, alpha 0.85. 미니멀(몸통 없음).
- **크기 가이드**: 날개폭 ≈ 36*s px. 작은 점 같은 새 2마리(scale 1, 0.8).
- **가독성**: y≈0.09~0.13(밝은 하늘) → 회색 실루엣 대비 충분.

### 4.4 `sandcastle` — 모래성 (beach)
- **앵커**: (x,y) = 성 바닥 중심(모래밭에 앉음).
- **구성**: 사다리꼴 본체(아래 폭 `40*s`, 위 폭 `30*s`, 높이 `26*s`, `0xddb878` 모래색). 좌우·중앙 작은 탑 3개(폭 `8*s`, 본체 위로 `12*s` 솟음, 같은 색). 각 탑 꼭대기 작은 삼각 깃발(`0xff6f61`, 폭 `7*s`). 본체에 아치형 문 1개(`0xbf9a5e` 어두운 모래, 반원). 윤곽 옅게 `0xc9a06a` width `1.5`.
- **크기 가이드**: 전체 폭 ≈ 40*s, 높이 ≈ 40*s px. parasol(높이 ~34*s)과 비슷한 톤.
- **가독성**: y≈0.94 하단 모래 surround. 모래색 본체+코랄 깃발이 sand 배경 위로 살짝 떠 보임.

### 4.5 `starfish` — 불가사리 (beach)
- **앵커**: (x,y) = 중심.
- **구성**: 5각 별(다섯 꼭짓점, 외반경 `14*s`·내반경 `6*s` 폴리곤), `0xff9f5a`(산호 주황) fill. 중앙에 옅은 점무늬 3~4개(`0xffd0a0`, 반경 `1.5*s`). 윤곽 `0xe07f3a` width `1.5`. 살짝 회전(약 `-0.3rad` 기울여 자연스럽게)은 렌더러 재량.
- **크기 가이드**: 별 지름 ≈ 28*s px. 작은 소품.
- **가독성**: 하단 모래밭. 주황 별이 sand 위로 또렷.

### 4.6 `beachball` — 비치볼 (beach)
- **앵커**: (x,y) = 공 중심.
- **구성**: 흰 원(반경 `16*s`, `0xffffff`). 컬러 웨지 6조각(중심에서 방사, 교대로 칠함) — `[0xff5a5a(빨), 0x4fa3e0(파), 0xffd64a(노)]` 순환, 각 웨지는 인접 두 각도 사이 부채꼴. 윤곽 `0xddddee` width `1.5`. 좌상단 하이라이트 작은 흰 원(`5*s`, alpha 0.6).
- **크기 가이드**: 지름 ≈ 32*s px. tube(반경 22*s)와 비슷.
- **가독성**: 바다 인필드 위. 밝은 원색이 터쿼이즈 위로 강한 대비.

## 5. 역할/원칙 점검

- 순수 시각: 팔레트·decor·ambient만 변경. 엔진/결정론/pickArena 레지스트리 무영향.
- 단순함: 테마당 신규 소품 3개(grassland: bunting/flower + sun·cloud 재사용 / beach: seagull·sandcastle·starfish·beachball). 과밀 회피 위해 인필드 중앙 안전존·상단 하늘·하단 모래 surround 3구역에만 배치.
