범위: partmodel 아트만. runStyle 변경 없음(개·고양이 gallop, 독수리 glide 유지). 엔진/렌더러 로직 무변경. 고슴도치 손대지 않음.

> 회전 단위: partmodel `rot`은 도(degree). gallop/glide의 다리·날개 스윙은 렌더러가 절차적으로 구동(±수십 도). pivot은 회전/스쿼시 중심일 뿐 도형의 절대 좌표는 그대로.

## 1. 🐶 개 (src/data/partmodels/dog.ts)
- **머리/눈 치비 강화**: 두개골 반지름 36→39, 눈 12→13r·하이라이트 4.8→5.4r(더 동그랗고 큰 눈). 코·머즐 퍼프 살짝 키움.
- **귀 가독성**: earL pivot을 위로(-18→-20)·길쭉한 물방울 실루엣으로 교체 + 안쪽 cheek 음영. 갤럽 시 펄럭임이 또렷.
- **꼬리 가독성**: 살짝 더 큰 컬 아치 + cheek 하이라이트 라인. 살랑살랑 wag가 읽힘.
- **다리 스윙**: 렌더러 gallop 분기가 ±42° 절차 스윙으로 구동(데이터 run 포즈는 비움 유지). run 포즈에 정적 rot을 넣으면 좌/우 비대칭으로 스윙 중심이 틀어져 더 부자연 → 데이터 변경 안 함이 정답.

## 2. 🐱 고양이 (src/data/partmodels/cat.ts)
- **뾰족 귀**: 삼각형 더 높고 날카롭게(-54→-60 꼭짓점), 안쪽 핑크도 동반 확대.
- **동그란 눈(도도함 유지)**: 아몬드 눈 8x9→9x10.5, 하이라이트 키움, 윗눈꺼풀 라인 더 또렷(2→2.2). 베이비감↑ + 새침 유지.
- **가는 꼬리**: S컬을 더 가늘게 테이퍼 + 끝 후크 + stripe 디테일. 또렷한 고양이 꼬리.
- **등선 다듬기**: body를 타원→커스텀 패스로. 뒤쪽 엉덩이가 약간 높은 가벼운 아치 등선(고양이다움). breast point 패치도 정리.
- **catwalk(skill) 포즈 도도하게**: `skill: head dy -5/rot -6, body dy -2, tail rot -30(높이 든 컬), frontLegL rot 26(앙증 들린 앞발), earL rot -8`. 턱 들고·꼬리 치켜세우고·앞발 살짝 든 캣워크.

## 3. 🦅 독수리 (src/data/partmodels/eagle.ts) — 가장 애매했던 핵심
- **머리/부리/브로우 날카롭게**:
  - crest(목덜미 볏)를 더 raked한 스파이크로.
  - 어두운 crown cap이 nape까지 내려와 밝은 cheek와 대비 또렷.
  - **브로우 릿지**를 각진 사다리꼴로 눈 위에 더 돌출(사나운 핵심 큐).
  - 눈을 브로우 아래로 더 파고들게 + 샤프 하이라이트.
  - **부리**: 둥근 곡선→직선 매스가 끝에서 후크로 꺾이는 더 날카로운 갈고리 + gape 라인 추가.
- **몸통 대비**: torso 실루엣을 앞가슴 높고 꼬리로 테이퍼되게 정리, 밝은 breast 패치 또렷.
- **날개 윤곽 가독성↑**: wingL을 어깨에서 시작해 층진 primary 깃 끝(노치)으로 테이퍼되게 재작도 + covert 솔기선.
- **다리접기 디테일**: 근/원 다리 발을 곡선 3-탈론으로 또렷하게. (글라이드 중 렌더러가 dy-12·rot로 배 밑에 접음.)
- **꼬리**: rectrices 사이 노치를 더 또렷이 + 깃 분할선.

### ⚠️ eagle 날개 파트 구조 (renderer-dev 펄럭임용 — 중요)
- 날개 파트는 **단일 근측 날개 `wingL` 하나** (wingR 없음). z=3.
- **pivot 변경: `{x:2,y:46}` → `{x:-4,y:44}` (어깨 위치로 이동)**. 이게 렌더러가 rot로 펄럭일 때 어깨를 축으로 자연스럽게 휘게 하는 핵심. 도형 절대좌표는 그대로라 위치는 안 변하고 회전 중심만 어깨로 옴.
- **기본 rot = 0** (idle/run 포즈에 wingL 델타 없음 → 안정적). 펄럭임은 0을 중심으로 rot 진동시키면 됨.
- 현재 글라이드 분기(PartsCharacter.ts L328-329): `if (name==='wingL') rot += sin(clock*4.2)*5` (소프트 플러터). renderer-dev가 여기 진폭/속도를 키워 "우아한 활공+간헐 펄럭"으로 업그레이드 예정. wingL/pivot(-4,44)/base rot 0 계약 유지하면 데이터 영향 없음.
- 참고 포즈 델타(어깨 축 기준 그대로 동작): skill `wingL rot -16`, win `-22`, fall `+24`.
