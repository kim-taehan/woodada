## 이번 배치 (먼저 시작)

### 1. maxFrames 자동 스케일 ✅
- **문제**: `simulateRace`의 기본 `maxFrames = 60*120`(120초)이 10바퀴 경주엔 부족해 "race did not finish within maxFrames"로 *조용히 throw*. (멀티랩 테스트 때 수동으로 24000을 넘겨야 했음.)
- **할 일**: `maxFrames`를 `laps`(+여유)에 비례해 자동 산정. 호출부가 매직넘버를 안 넘겨도 어떤 바퀴 수든 완주. 결정론·기존 동작 불변.
- 영역: engine · 리스크 낮음

### 3. 멀티랩 공정성 게이트 ✅
- **문제**: `engine-bias` 테스트가 **1바퀴 전용** → 10바퀴 밸런스가 사각지대였음(수동으로 발견·수정).
- **할 일**: 공정성 게이트를 `laps ∈ {1, 3, 10}`로 파라미터화(느슨한 기준 유지 — "모두 이길 수 있고 아무도 독주 안 함"을 각 바퀴 수에서). 장기전 밸런스 회귀를 자동 검출.
- 영역: test(engine-bias) · 리스크 낮음

### 4. 밸런스 하니스 정식화 ✅
- **문제**: 밸런스 측정을 매번 임시 스크립트로 수동 실행.
- **할 일**: 개인/팀/릴레이 × 바퀴수(1/3/10) 매트릭스를 한 번에 출력하는 도구로(`scripts/balance.ts` 확장 또는 신설). 등수분포·1등%·독주지표 포함.
- 영역: tooling(scripts) · 리스크 낮음
