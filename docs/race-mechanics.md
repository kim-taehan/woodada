# 우다다 경주 메커니즘 (시스템 요소)

> 스킬·아이템처럼 캐릭터 개별 효과는 제외하고, **모든 경주에 공통으로 작용하는 시스템 규칙**만 정리한다.
> 엔진은 순수 시뮬레이션이다 — 화면/트랙 모양을 모르고, 추상 좌표 **progress(전후 거리)** 와 **lane(0=인코스 끝 … 1=아웃코스 끝, 연속값)** 만 다룬다. 같은 (설정 + 시드)는 **항상 같은 경주로 재생**된다(결정론). 모든 무작위성은 시드 RNG.

---

## 1. 트랙 구간 — 직선 / 곡선

오벌(스타디움) 한 바퀴를 랩 위치 `u`(0~1)로 보면 네 구간이다 (출발선 = 바닥 직선 왼쪽 시작, 반시계):

| 구간 | 랩 위치 u | 비중 |
|---|---|---|
| 바닥 직선 | 0.00 ~ 0.28 | |
| 우 곡선 | 0.28 ~ 0.50 | |
| 상단 직선 | 0.50 ~ 0.78 | |
| 좌 곡선 | 0.78 ~ 1.00 | |
| **직선 합** | | **0.56** |
| **곡선 합** | | **0.44** |

엔진은 이 구간 경계를 **고정 상수**로 알고(렌더러 오벌과 정합), 아래 두 가지가 구간에 따라 달라진다: **인코스 거리 이점(§3)** 과 **캐릭터 코너링 속도(§5)**.

---

## 2. 속도 구성

매 프레임 레이서의 속도는 다음을 곱·합해 만든다:

```
speed = (baseSpeed + 코너링바이어스) × jitter × condition  (+ 스킬 burst)
speed = speed × catchupFactor          (러버밴딩)
```

- **baseSpeed** — 경주 시작 시 레이서마다 **랜덤 1.3~1.5** 배정(운). + `speed` 스탯 미세 보정(`speedGain 0.018`, 거의 무의미).
- **jitter** — 매 프레임 ±8% 노이즈.
- **condition** — 매 랩 폼(§4).
- **catchupFactor** — 평균 진행도로 끌어당기는 러버밴딩(§6).
- (코너링·condition·catchup이 주된 속도 변동원. `speed` 스탯 자체 차이는 사실상 무시 가능.)

---

## 3. 레인 = 거리 (인코스 우위) — 곡선 한정

인코스(안쪽)는 **곡선에서만** 호(arc)가 짧다. 그래서 거리 페널티가 **곡선에서만** 걸린다:

```
progress += speed × laneDistanceFactor(lane, 구간)
laneDistanceFactor = 1 − distLoss × lane   (곡선)   /   1   (직선)
distLoss = 0.12   (아웃코스 끝은 곡선에서 12% 손해)
```

- **곡선**: 인코스가 빠름 → 안쪽 다툼.
- **직선**: 레인 무관(모든 레인 길이 같음) → **바깥으로 추월해도 손해 0 = 자유 추월 구간**.
- `progress`는 이렇게 "실제 달린 거리(보정 진행도)"가 되어, 순위·결승·데스매치 판정이 레인 무관하게 공정하다.

---

## 4. 컨디션 (매 랩 폼)

레이서가 **랩 라인을 통과할 때마다** 자기 시드 스트림에서 **1~10**을 굴려 그 랩의 속도 배율을 정한다:

```
condition = 1 + (roll − 5.5) × 0.03      (roll 1~10)
→ roll 1  ≈ ×0.865 (나쁜 폼)   …   roll 10 ≈ ×1.135 (좋은 폼)
```

- 중앙값(5.5) 기준이라 **한 바퀴 평균은 ±0 → 승률 공정성 유지**(좋은 랩·나쁜 랩이 상쇄).
- **같은 동물 여러 마리도** 매 랩 폼이 갈려 순위가 뒤집힌다 + 추첨 게임의 **운 요소**.
- 세기 조절: `CONDITION.gain`(현 0.03).

---

## 5. 캐릭터 코너링 특화 (직선형 / 곡선형)

캐릭터마다 `cornering` 스탯(1~5, 중앙 3)으로 **구간별 속도**가 갈린다:

- **낮음(예 dog=1)** = 직선 스프린터: 직선에서 빠르고 곡선에서 느림.
- **높음(예 spider=5)** = 곡선 스페셜리스트: 그 반대.

```
곡선   바이어스 = +corneringGain × dev × 직선비중(0.56)
직선   바이어스 = −corneringGain × dev × 곡선비중(0.44)
corneringGain = 0.35,  dev = (cornering−3)/2
```

- 한쪽 구간의 가속을 **반대 구간 길이로 가중**해 상쇄 → **한 바퀴 평균 = ±0, 공정성 유지**. 어디서 빠른지만 달라진다.
- 효과: **직선에선 직선형이, 곡선에선 곡선형이 앞서** 매 구간 전환마다 리드가 바뀐다(순위 변동).

---

## 6. 캐치업 (러버밴딩, 독주 방지)

평균 진행도 기준으로 뒤처지면 가속, 앞서면 감속:

```
behindGain 2.6 (뒤처진 만큼 가속) / aheadDrag 2.2 (앞선 만큼 감속)
배율 한계 minBoost 0.8 ~ maxBoost 1.2
```

- 팩을 뭉쳐 두고 리드가 자주 바뀌게 한다. 대형 필드에선 꼴찌 가속을 약화해 앞뒤로 늘어지게 함(독주 없이 스트링).

---

## 7. 레인 AI — "우당탕탕 난투 스크램블"

라인을 지키는 주행이 아니라, **모두 더 빠른 인코스를 두고 엉켜 다투는** 모델. lane은 매 프레임 여러 영향을 **하나의 목표로 합산 → 한 번의 제한된 드리프트**(서로 안 싸워 지터 없음).

**두 가지 레인 변경 규칙:**
1. **추월** — 바로 앞에 더 느린 레이서가 막으면: **안쪽 우선**(거리도 이득) 시도 → 막혔으면 **바깥쪽**으로 우회(직선이면 손해 0). 더 빠르지 않으면 추월 안 하고 뒤에서 감속(큐잉).
2. **거리 단축** — 경합 없으면 **인코스(레일)로** 붙는다.

**혼잡도 분산(scramble)** — 모두 인코스로 몰리면 한 줄이 되므로, **앞·안쪽이 막힌 정도에 비례해 바깥으로 비집어** 팩을 폭 전체로 퍼뜨린다(빈 레인이 생겨 스턴/정체도 비켜 감).

주요 값: `innerGoal 0.05`(레일), `laneStep 0.3`(추월 폭), `laneDrift 0.05`(프레임당 이동), `scrambleGain 0.25`, `weaveHoldFrames 12`(추월 중 방향 유지).

---

## 8. 충돌 — 개인 영역 (통과 금지)

각 레이서는 **개인 버블**을 가져 서로 못 뚫고 지나간다:

- **앞뒤**: 같은 레인밴드에서 뒤 레이서는 앞 레이서의 `progress − minGap`을 못 넘는다(클램프). `minGap 12`(≈1몸길이). 부스트로도 통과 불가 → **추월하려면 돌아가야** 한다.
- **감지/반응**: `nearAhead 16`(버블 앞에서 미리 감속·위빙), `laneNear 0.16`(같은 레인 판정 폭).
- 막혀서 우회 못 하면 감속(`blockDecel 0.5`, 파워 높으면 덜 감속).

---

## 9. 공정성 (불변 규칙)

- **결정론**: 같은 (설정+시드) = 같은 경주. 골든 스크린샷·단위테스트가 의존.
- **engine-bias 게이트**: 여러 경주 통계에서 **모든 캐릭터·모든 출발 슬롯이 이길 수 있고, 아무도 독주하지 않는다**(1·3·10랩).
- 위 속도 변동원(코너링·컨디션)은 전부 **한 바퀴 평균이 0이 되게 설계**되어 역동성을 더하면서 공정성을 깨지 않는다. 출발 위치·레인은 결과를 좌우하지 않는다(인코스 잠금 없음 — 레이서가 실제로 인코스를 두고 경쟁).
